## 도모다치 채팅 앱 기획서 (초안)

### 1. 서비스 개요

- **서비스명(가칭)**: 도모다치 채팅
- **한 줄 소개**: 친한 친구/지인들과 소규모로 깊이 대화할 수 있는, 깔끔하고 직관적인 실시간 채팅 서비스.
- **핵심 가치**
  - **집중된 소통**: 소규모 그룹과 1:1에 최적화된 경험
  - **간결한 인터페이스**: 방해 요소를 최소화한 미니멀 UI
  - **신뢰 기반**: 닉네임/프로필 중심의 가벼운 정체성 + 최소한의 개인정보

### 2. 주요 타깃 및 사용 시나리오

- **타깃 사용자**
  - 친구/지인들과 사적인 소규모 커뮤니티를 만들고 싶은 20~30대
  - 사이드 프로젝트/스터디/작은 팀 단위로 빠르게 소통하고 싶은 사용자

- **주요 사용 시나리오**
  - 1:1 대화: 친구와 실시간으로 대화, 사진/링크 공유
  - 소규모 그룹 채팅: 스터디, 소모임(3~10명) 중심의 그룹 채팅
  - 알림 기반 복귀: 중요한 멘션/답장을 보고 앱으로 복귀

### 3. 핵심 기능 정의

- **회원/세션**
  - 이메일 또는 소셜 로그인(초안 단계에서는 이메일+Magic Link 또는 간단한 로그인 방식 가정)
  - 프로필 설정: 닉네임, 한 줄 소개, 프로필 이미지(선택)

- **채팅 기본 기능**
  - 1:1 채팅
  - 그룹 채팅방 생성 / 참여
  - 메시지 전송(텍스트)
  - 메시지 상태: 전송됨 / 읽음(최소 1단계, 필요 시 추후 세분화)
  - 메시지 수정/삭제 (초기 버전에는 “삭제만” 지원 고려)

- **부가 기능 (우선순위 낮음, 후순위 개발)**
  - 이미지/파일 전송
  - 메시지에 이모지 리액션
  - 메시지 검색(방 단위, 전체 단위)

### 4. 정보 구조(IA) 및 화면 구성(초안)

- **상단 구조**
  - `앱 헤더`: 서비스 로고 또는 제목, 사용자 프로필 진입 버튼

- **좌측 패널 (데스크톱 기준)**
  - 채팅 리스트
    - 최근 대화방 목록
    - 각 방의 마지막 메시지, 안 읽은 메시지 카운트
    - 새 채팅/새 그룹 생성 버튼

- **우측 메인 패널**
  - 선택한 채팅방 뷰
    - 상단: 방 이름, 멤버 수, 옵션 버튼(방 나가기 등)
    - 중앙: 메시지 리스트(스크롤)
    - 하단: 메시지 입력창(텍스트 입력, 전송 버튼)

- **모바일 고려**
  - 채팅 리스트 / 채팅방을 탭 전환 또는 슬라이드 네비게이션으로 분리
  - 하단 탭바(예: `채팅`, `프로필`) 최소 구성

### 5. UX 원칙

- **단순함**
  - 필수 기능만 먼저 노출, 고급 기능은 후순위
  - 1~2번의 탭으로 주요 행동(메시지 보내기, 방 이동)이 가능하도록 설계

- **가독성**
  - 말풍선 정렬(내 메시지 우측, 상대 메시지 좌측)
  - 날짜/시간 구분을 시각적으로 명확히

- **피로도 감소**
  - 과도한 애니메이션, 알림 최소화
  - 방 입장 시 최근 메시지부터 자연스럽게 스크롤

### 6. 기술 및 아키텍처 방향(초안 관점)

- **프론트엔드**
  - Next.js 기반 웹 앱 (이미 프로젝트 기반 존재)
  - 실시간 통신: WebSocket 또는 서버 이벤트(SSE) 활용 검토

- **백엔드**
  - 인증: 세션 기반 또는 JWT 기반 간단한 인증 계층
  - 채팅 메시지 저장: RDB 또는 NoSQL (초안 단계에서는 세부 기술 스택 미정, “메시지/유저/채팅방” 기본 스키마 전제)

- **실시간 요구사항**
  - 방별 메시지 브로드캐스트
  - 온라인/오프라인 상태는 1차 버전에서는 생략하거나 단순 표시만 고려

### 7. 비기능 요구사항

- **성능**
  - 메시지 리스트 가상 스크롤링 적용(메시지가 많아질 경우를 고려)
  - 모바일 환경에서도 부드러운 스크롤과 입력 경험

- **보안/프라이버시**
  - 기본적인 TLS(HTTPS) 통신 전제
  - 최소한의 개인정보 수집, 로그아웃/세션 만료 정책

- **확장성(향후)**
  - 향후 음성/영상, 파일 공유 기능을 추가할 수 있도록 도메인/데이터 구조 설계

### 8. MVP 범위 제안

- **MVP에 포함**
  - 회원 가입/로그인
  - 프로필 편집(닉네임, 이미지)
  - 1:1 채팅
  - 그룹 채팅 생성 및 참여
  - 실시간 텍스트 메시지 전송 및 읽음 처리(단순)

- **MVP 이후**
  - 이미지/파일 전송
  - 메시지 검색
  - 메시지 리액션, 고정 메시지, 공지 기능
